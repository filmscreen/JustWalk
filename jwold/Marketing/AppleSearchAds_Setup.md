# Just Walk - Apple Search Ads Setup Guide

## Getting Started

1. Go to https://searchads.apple.com
2. Sign in with your Apple Developer account
3. Accept terms and set up payment method

---

## Initial Settings (Conservative Start)

### Account Level
- Daily budget cap: $5-10/day to start
- Payment threshold: Set low ($100) initially

### Campaign Settings
- **Countries:** Start with US only
- **Age:** 18-65
- **Gender:** All
- **Customer type:** All users (not "Users of my other apps")

### Ad Group Settings
- **Default bid:** $0.50 (adjust based on results)
- **CPA goal:** $2.00 (cost per acquisition)

## Key Metrics to Watch

| Metric | Target | Action if Off |
|--------|--------|---------------|
| Tap-through rate (TTR) | >5% | Improve screenshots/description |
| Conversion rate (CR) | >40% | Improve App Store page |
| Cost per acquisition (CPA) | <$2 | Lower bids or pause keywords |
| Cost per tap | <$0.30 | Adjust bids |

---

## Weekly Optimization Checklist

- [ ] Check which keywords have impressions but no taps (improve relevance or pause)
- [ ] Check which keywords have taps but no downloads (landing page issue)
- [ ] Review Search Match discoveries for new keyword ideas
- [ ] Adjust bids on high-performing keywords (increase by $0.10)
- [ ] Pause keywords with CPA > $3 after 50+ impressions
- [ ] Add negative keywords for irrelevant searches

## Budget Scaling Guide

### Month 1: $150-300
- Learn which keywords convert
- Establish baseline CPA
- Test different match types

### Month 2-3: $300-500 (if positive ROI)
- Scale winning keywords
- Add feature-focused campaigns
- Create custom product pages

### Month 4+: Based on LTV
- Calculate lifetime value of Pro subscribers
- Scale budget where CPA < LTV
- Consider 30-40% of revenue for ads
